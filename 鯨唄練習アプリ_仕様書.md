# 鯨唄練習アプリ 仕様書
## Claude Code 引き継ぎ用ドキュメント

---

## 1. プロジェクト概要

鯨唄（くじらうた）の自習用練習アプリ。先生から提供されたMP3音源を基準に、リアルタイムで自分の声と比較しながら練習できるカラオケ採点風Webアプリ。

- **形式**: 単一HTMLファイル（MP3をbase64で埋め込み済み）
- **動作環境**: PC・スマホのブラウザ（Safari / Chrome）。オフライン動作可
- **現在の最新版**: `鯨唄練習アプリv9.html`

---

## 2. 背景・ユーザー情報

- 鯨唄教室に通っている初学者
- 先生と数回一緒に歌ったが自主練習がほとんどできていない
- モチベーション維持のためカラオケ的な採点機能が欲しい
- スマホ（古いiPhone SE）での使用を想定
- Google Drive経由でHTMLファイルをスマホに転送して使用

---

## 3. 楽曲情報

- **曲名**: 祝いめでた
- **音源**: 先生のMP3（`01_トラック_1祝いめでた.mp3`）をbase64でHTMLに埋め込み済み
- **鯨唄の特性**: 音符なし・自由なテンポ・独特の節回し。「正解の音程」は存在せず、先生の節回しの「形の近さ」で評価する

## 5. 機能仕様

### 5.1 STEP 1 – 先生の音源再生・解析

- HTMLに埋め込まれたMP3を `b64ToArrayBuffer()` でデコードしてBlob URLを生成し `<audio>` にセット
- 「先生の音程を解析する」ボタン押下で `OfflineAudioContext` を使いオフライン解析
- `fetchは使用禁止`（base64 → ArrayBuffer → decodeAudioData の流れで処理）
- フレームサイズ: 2048、ホップサイズ: 1024
- `teacherHopSec = hopSize / sampleRate`（≈0.023秒/フレーム）
- 音域の自動検出: 有効ピッチの1〜99パーセンタイルを取り、上下8〜12%マージンを加えて `teacherMinF` / `teacherMaxF` を設定
- 移動平均（ウィンドウ20フレーム）で `teacherSmooth` を生成
- 解析後すぐに先生の軌跡をグラフに表示

### 5.2 流れる比較グラフ

- **背景色**: `#12122a`（ダーク）
- **グラフサイズ**: canvas height=500px（CSS幅100%）
- **表示範囲**: 全体フレーム数の約4%を画面に表示（波がゆっくり見える＝3倍ズーム）
- **現在位置**: 画面左から22%の縦点線
- **流れ方**: 右から左へ（過去が左、未来が右）
- **先生ライン（黄色 `#fbbf24`）**: 実ライン（lineWidth=3）＋移動平均帯（lineWidth=18、透明度20%）
- **自分のライン**: 表示しない（球のみ）
- **グリッド**: 横線4本＋音名・Hz表示（左端）
- **歌詞**: グラフ下部に黄色（`rgba(255,210,80,0.8)`）で流れる

### 5.3 玉（ボール）の表示

- **先生の玉（黄色 `#fde68a`）**: 半径13px、白枠、グロー効果
- **自分の玉（水色 `#93c5fd`）**: 半径13px、白枠、グロー効果
- 両方とも同じ**EMA（指数移動平均）平滑化**: `BALL_SMOOTH = 0.08`
  - `smoothY = smoothY + (targetY - smoothY) * 0.08`
  - 低いほど動きが遅く安定する
- 声が検出されない場合: 自分の玉は消える（`smoothMyY = null`）
- 先生・自分の玉は両方とも現在位置の縦線上に表示

### 5.4 音程検出（自己相関法 ACF）

```javascript
function autocorrelate(buf, sampleRate) {
  // RMS閾値: 0.006（これ以下は無音と判断）
  // 相関係数閾値: 0.86
  // 検出範囲: 40Hz〜3000Hz

```

- `analyser.fftSize = 1024`（低レイテンシ）
- `analyser.smoothingTimeConstant = 0`（スムージングなし）
- マイク設定: `echoCancellation: false, noiseSuppression: false, autoGainControl: false`
- ライブ表示用バッファ: 4フレームの短い移動平均（`LIVE_BUF = 4`）

### 5.5 録音・再生同期

- 「録音開始」ボタンで先生音源を `currentTime=0` から自動再生
- `scrollPos = Math.floor(refAudio.currentTime / teacherHopSec)` で再生位置同期
- `requestAnimationFrame` ループで毎フレーム更新

### 5.6 一致度表示

- 先生ピッチと自分ピッチを時間比率でマッピングして比較
- 半音差 < 2 → 1点、< 4 → 0.5点、それ以上 → 0点
- 0〜100%で表示

### 5.7 採点（唄い終わり後）

4項目の加重平均でスコアを算出（0〜100点）：

| 項目 | 重み | 計算方法 |
|------|------|----------|
| 節回しの近さ | 40% | 上記一致度 |
| 音量・強弱 | 20% | 平均dBから換算 |
| 継続・伸び | 20% | 録音時間から換算（基準30秒） |
| 声の安定 | 20% | ピッチ変化量から換算 |

ランク：S（90〜）/ A（78〜）/ B（65〜）/ C（50〜）/ D（〜49）

### 5.8 練習記録

- セッション内メモリに蓄積（ページリロードでリセット）
- 日時・スコア・ランクを一覧表示

## 7. 技術スタック

- **言語**: HTML / CSS / Vanilla JavaScript（フレームワークなし）
- **Web API**: Web Audio API（AudioContext, OfflineAudioContext, AnalyserNode）、MediaDevices API、Canvas 2D API
- **依存ライブラリ**: なし（すべてブラウザネイティブ）
- **音源埋め込み**: base64文字列をHTML内にハードコード

## 8. 既知の問題・注意点

- `fetch()` でbase64 data URIを読もうとすると失敗する（特にiOS Safari）。必ず `b64ToArrayBuffer()` → `decodeAudioData()` の流れで処理すること
- 歌詞定数 `LYRICS` は `total`（フレーム総数）が定義された後でのみ参照可能。描画関数内で `total` の定義より前に置くと `Cannot access 'total' before initialization` エラーになる
- iOSではマイク許可ダイアログへの対応が必要
- ページリロードで練習記録はリセットされる（永続化未実装）
